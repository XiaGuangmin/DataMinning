# 科技盘后扫描 (stock_scan)

科技主线股票池（固定池 80 只 + 候选池，按 9 个细分方向分类）的盘后扫描脚本。
**纯本地运行，无需联网服务器、无需 GitHub**，在自己电脑上装好 Python 依赖即可跑。

## 为什么用这套数据源

原实现依赖东方财富 `push2.eastmoney.com` / `push2his.eastmoney.com` 接口，
该接口对部分出口 IP 会返回 502/超时导致取数失败。本脚本改用
**腾讯财经为主、新浪财经为备**的数据源（实测稳定），并把整条流水线固化为脚本，
每次运行口径一致、可复现。

## 环境要求

- Python 3.9+（用到 `zoneinfo`；Windows 若报时区错误，脚本会自动退回 UTC+8，无需额外操作）
- 依赖：`openpyxl`（生成 Excel）

## 用法

```bash
cd stock_scan
pip install -r requirements.txt
python3 scan.py            # Windows 用 python scan.py
```

- 非交易日（当日无 K 线）：打印 `NON_TRADING_DAY` 并正常退出，不产出文件。
- 回补/测试指定日期：
  - macOS/Linux：`SCAN_DATE=2026-07-07 python3 scan.py`
  - Windows PowerShell：`$env:SCAN_DATE="2026-07-07"; python scan.py`

  （指定日须为交易日。）

## 每天自动跑（可选，在你本机上设置）

- **macOS / Linux（cron）**，工作日北京时间 15:30：
  ```
  30 15 * * 1-5  cd /路径/stock_scan && /usr/bin/python3 scan.py >> run.log 2>&1
  ```
- **Windows**：用「任务计划程序」新建任务，触发器设为工作日 15:30，
  操作运行 `python`，参数 `scan.py`，起始位置填 `stock_scan` 目录。

## 产出（output/ 目录）

- `scan_<日期>.xlsx`：4 个 Sheet
  1. 全量数据（全部个股换手率/量比/涨跌幅/近5日涨幅/均线状态/信号标记）
  2. 信号汇总与操作建议（启动/加速个股 + 技术指标状态 + 操作建议）
  3. 技术指标监测（信号个股 MACD/DIF/DEA/KDJ/RSI/买卖点/共振）
  4. 板块异动分析（各细分方向平均涨幅、上涨家数、领涨前三、异动标记）
- `summary_<日期>.md`：≤2000 字关键摘要（启动信号、加速清单、板块异动、指标共振）。

## 数据源

| 用途 | 主源 | 备源 |
|------|------|------|
| 实时/收盘行情（换手率、量比） | `qt.gtimg.cn`（腾讯） | `hq.sinajs.cn`（新浪） |
| 日 K 线（前复权，算技术指标） | `web.ifzq.gtimg.cn`（腾讯） | — |

腾讯行情单批最多 50 只、K 线单只一请求（96 只压力可忽略）。行情主源失败或返回不足
80% 时自动切换新浪备源（新浪无换手率/量比，量比改由 K 线成交量近似计算）。

> **运行时机很重要**：`涨跌幅`、`近5日涨幅`、`板块异动`、全部技术指标均来自**日 K 线**
> （带日期，可复现）；但 `换手率`、`量比` 来自**实时行情快照**，只反映"此刻"。
> 因此本脚本应在**交易日收盘后（北京时间 15:00 之后，建议 15:30）运行**，此时实时快照
> 已等于当日收盘值，换手率/量比才准确、启动/加速信号才有效。
> `SCAN_DATE` 回补历史日期时，K 线相关字段准确，但换手率/量比无法还原（会反映运行当下），
> 故回补主要用于验证板块与技术指标口径，不用于还原当日启动/加速信号。

## 筛选与指标口径

- **启动信号**：换手率 3%–8% 且量比 1.5–3 且近 5 日涨幅 5%–15%，并满足以下之一：
  横盘（前 20 日振幅≤10%）后突破 / 刚站上 20 日线 / 刚站上 60 日线。
- **加速警告**：换手率 >15% 记红色；换手率 10%–15% 或量比 3–5 或近 5 日涨幅 15%–25% 记黄色。
- **技术指标**（全部 96 只均取 60+ 日 K 线计算，信号个股写入监测表）：
  MACD 金叉/死叉（区分零轴上下方）与底/顶背离（强/中/弱）、KDJ 超卖金叉/超买死叉、
  RSI 超卖反弹/超买回落；≥2 个买点或卖点指标同时触发标记为强买/卖点共振。

## 候选池机制

`pool.json` 的 `candidate_pool` 持久化候选股及其 `status`（confirmed / observing）与
`added_date`。板块异动（某方向平均涨幅≥阈值）时可在该板块内按启动标准补入候选股
（新入池标 `observing` 并记 `added_date`）；观察满 3 个交易日后由脚本给出晋升/移出建议。
状态直接写回本地 `pool.json`，保证跨天不丢失。

## 配置

编辑 `pool.json`：`fixed_pool` 调整固定池，`candidate_pool` 维护候选池，
`meta.sector_alert_threshold_pct` 调整板块异动阈值，`meta.candidate_max` 为候选池上限。
